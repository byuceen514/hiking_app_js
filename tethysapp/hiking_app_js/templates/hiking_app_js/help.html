Use Guide and Help Documentation

The following sections include the User Guide and Technical documentation for the Hiking Trail Explorer App.

User Guide

Overview

The purpose of the “Hiking Explorer” tool is to allow users to draw a desired hiking path in the state of Utah and view the elevation gains and losses. The user draws the path on a basemap by clicking and drawing a polyline. The map also displays Utah County lines in addition to existing trails. The tool is located in a Tethys based web application.

Accessing Website

The “Hiking Explorer” web app can be accessed on the BYU server. This web-site is intended to be an educational tool, for viewing elevation gains and losses for any possible trail or route in the state of Utah.

The “Hiking Explorer” app should be compatible with Internet Explorer, Mozilla Firefox, and Google Chrome.

Using the Website

The following figures show the various elements of the “Hiking Explorer” web interface. The key elements of the user interface include the map interface and navigation buttons.



The map interface window allows the user to manipulate the map’s extents. Left-click and dragging the mouse will allow the user to pan the map’s display. The mouse scroll wheel, or the “+” and “-” buttons on the map interface, allows the user to zoom in and out of the map.

The user can draw a polyline or free hand polyline by first clicking on either of the buttons below the “Draw:” text. This polyline should represent a trail the user is interested in. Once the user is done drawing the polyline, an elevation profile showing elevation (ft) vs distance (miles) will appear above the interactive map.

The navigation pane allow the user to go to the Homepage, the Hiking Tool, the Help Documentation, as well as various useful links.

Technical Documentation

Technical Specifications

The Hiking Trail Explorer App was developed as part of the GIS Programming course taught through the Civil and Environmental Engineering Department at Brigham Young University (BYU). It was built using Tethys Platform, a web development framework created at BYU that simplifies the process of creating interactive geospatial web apps. Though Tethys Platform provides a basic design template and framework, the content, layout, and interactive elements unique to the Hiking Trail Explorer were co-created by Jasmyn Harper and Samuel Mineer, with help from Sarva Pulla and Dr. Dan Ames.

Installation and Setup

The Tethys Platform, and thus the Hiking Trail Explorer app, must be set up to run on a Linux operating system. Linux, as an open-source operating system, is free to obtain and install. If a physical machine is not available to run Linux as its native operating system, virtual machine programs, such as VirtualBox, are available for free and can be downloaded and installed on both Windows and Mac operating systems.

Once one has access to a Linux operating system, Tethys can be installed by following the instruction given on the Tethys Documentation Page. Another option is to install a Linux virtual image that has Tethys already pre-configured. This image can be obtained by contacting Dr. Daniel Ames at Brigham Young University.

Once Tethys has been installed, the source code for Hiking Trail Explorer app can be obtained and downloaded from GitHub. Once the source code is stored locally on your computer, the command line must be used to run the setup.py script that is located in the root folder with the “develop” parameter (“python setup.py develop”). This command must be run while in the Tethys python environment (“. /usr/lib/tethys/bin/activate”).

Finally, the server can be started (“tethys manage start”), and you can navigate to the Tethys Apps Library, which is defaulted to port 8000 of the localhost. If everything was done correctly, you should see the Tethys Apps Library page with an icon for the Hiking Trail Explorer. This icon can then be clicked to start the app.

User Interface Overview

The Hiking Trail Explorer web app is an interactive mapping environment where the data being displayed on the page is updated based on the user’s actions. The only action available to the user is using the “Polyline” and “Freehand Polyline” buttons to delineate and graph the profile of the hiking trail. These buttons allow the user to virtually draw a trial of interest, with existing trails in Utah and the county borders on the map. The resulting graph displays the elevation gains and losses and the approximate length of the proposed hiking trail.

How It Works

When the user selects either the “Polyline” or “Freehand Polyline” button, they are able to draw a line of interest for a hiking trail. Once the user has completed drawing the line, the line is sent to the geoserver, where the geoprocessing takes place. The geoserver will use to line drawn by the user to profile the length and elevation of the hiking trail, which creates a table of data points. These data points are sent back to the user from the server and a plot is created displaying the length and elevation of the hiking trail. This process is repeated for each new line drawn by the user with the “Polyline” or “Freehand Polyline” buttons.
